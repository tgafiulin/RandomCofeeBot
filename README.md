# Random Coffee (Telegram-бот)

Бот для сообщества **Random Coffee**: еженедельный опрос «участвую / нет», затем случайные пары и объявление результатов в группе. Расписание опроса и матчинга задаётся в **личке с ботом** (`/schedule`); внутри используется node-cron. Пояс по умолчанию — константа `DEFAULT_CRON_TIMEZONE` в `src/cronConstants.ts`; для отдельной группы можно задать **`/crontz`**.

Стек: **Node.js** (ESM), **TypeScript**, **grammY**, **Prisma** + **SQLite**, **node-cron**, long polling.

---

## Запуск

### Требования

- Node.js **18+** (рекомендуется актуальная LTS)
- Токен бота у [@BotFather](https://t.me/BotFather)

### Установка

```bash
npm install
```

Скопируйте переменные окружения и заполните их:

```bash
cp .env.example .env
```

Примените миграции базы (создаст/обновит SQLite-файл из `DATABASE_URL`):

```bash
npx prisma migrate deploy
```

В `prisma/migrations/` сейчас **одна** стартовая миграция `init` (актуальная схема целиком). Если у тебя в БД уже записаны **другие** имена миграций или менялся файл `init` после применения, Prisma может ругаться на checksum — тогда удали файл SQLite (или `npx prisma migrate reset`) и снова `npx prisma migrate deploy`.

Для разработки с автоперезапуском:

```bash
npm run dev
```

Сборка и запуск как в продакшене:

```bash
npm run build
npm start
```

После изменения схемы Prisma: `npx prisma migrate dev` (локально) или `npx prisma migrate deploy` (деплой), затем при необходимости `npx prisma generate`.

## Переменные окружения

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `BOT_TOKEN` | да | Токен бота |
| `DATABASE_URL` | да | SQLite; путь в `file:…` задаётся **от каталога `prisma/`** (рядом со `schema.prisma`), например `file:./dev.db` → файл `prisma/dev.db` |
| `ALLOWED_GROUP_IDS` | нет | Список id групп/супергрупп через запятую. **Не задан** — бот отвечает в любой группе (удобно для отладки). **Пустая строка** — группы игнорируются. **Задан список** — в группах работает только в перечисленных чатах |
| `ADMIN_TELEGRAM_IDS` | нет | User id админов через запятую: настройки cron в личке (`/settings`, `/schedule`, …), а также `/help` и подсказка `/chatid` **в личке** |

Пояс для cron без **`/crontz`** у группы задаётся в коде: `src/cronConstants.ts` → `DEFAULT_CRON_TIMEZONE`.

Команда **`/cron`** в личке показывает эквивалентное выражение cron для настроенного времени (формат: `минута час день_месяца месяц день_недели`; 0 или 7 — воскресенье). Подсказка: [crontab.guru](https://crontab.guru).

**Важно:** пока `ALLOWED_GROUP_IDS` пустой или не задан осмысленный список, автозапуск опроса/матчинга по cron не регистрируется — смотрите логи при старте.

## Использование

### Участники

В **группе** отвечают на **опрос кнопками**. Отдельные команды для этого не нужны. Команды бота в группе для обычных участников намеренно **не отвечают** (кроме сценариев ниже для админов).

### Администраторы в группе

В группе **имеют смысл** только:

- **`/help`** — краткая памятка уходит **в личку** боту (чтобы не светить инструкции всем в чате). Доступ: **создатель или администратор чата в Telegram** *или* пользователь из `ADMIN_TELEGRAM_IDS`.
- **`/chatid`** — **id этой группы** для строки `ALLOWED_GROUP_IDS` в `.env`; текст приходит **в личку**. Те же права, что у `/help`. Удобно для новой группы **до** добавления id в `.env` (обработчик зарегистрирован до фильтра whitelist).

Перед первым использованием `/help` и `/chatid` из группы админу стоит написать боту **`/start` в личке**, иначе личное сообщение может не доставиться — тогда бот кратко ответит **в группе** с подсказкой.

Остальные команды (`/start`, `/settings`, `/cron`, …) в группе **игнорируются** (без ответа).

### Настройка расписания (только личка)

Пользователи из **`ADMIN_TELEGRAM_IDS`** в личке с ботом:

1. **`/settings`** — выбрать группу из списка `ALLOWED_GROUP_IDS` (нужен непустой whitelist в `.env`).
2. **`/schedule`** — мастер: опрос или матчинг → день недели → время **`чч:мм`** сообщением.
3. **`/cron`** — сводка по выбранной группе.
4. **`/crontz Europe/Minsk`** — часовой пояс только для выбранной группы (иначе используется `DEFAULT_CRON_TIMEZONE` из кода).
5. **`/scheduleoff`** / **`/scheduleon`** — выключить или включить авто-опрос и авто-матчинг для выбранной группы.

### Личка: `/help` и пояснение `/chatid`

**`/help`** в личке отвечает только если ваш id есть в **`ADMIN_TELEGRAM_IDS`**.

**`/chatid`** в личке — только для тех же id: напоминание, что id **группы** нужно запросить командой **`/chatid` уже из самой группы**.

## Как это устроено кратко

- Раунд: опрос → ответы сохраняются → по cron или после ручного сценария вызывается матчинг и итог в группу.
- Один открытый раунд на чат; при совпадении cron с уже открытым опросом запуск может быть пропущен (см. логи).
